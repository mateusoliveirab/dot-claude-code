# User Instructions

## Core Rules

1. **Ask first, act later** — Never assume. Clarify doubts before implementing.
2. **Suggest, don't execute** — Discuss options before taking action.
3. **Stay focused** — Only do what was asked. Suggest improvements separately.
4. **Be direct** — Concise, objective responses.

## Dev Environment

- Dev server is usually already running — don't start it, test directly on localhost.
- Use `git` CLI (not `gh` CLI) for version control.
